# Streamlit项目开发框架 - 目录结构设计文档

## 1. 项目目录结构

```
my_project_name/          # 项目根目录
├── .streamlit/           # Streamlit配置
├── assets/               # 静态资源
│   └── images/           # 图片资源
├── data/                 # 数据文件
├── database/             # 数据库文件（新增）
├── logs/                 # 日志文件
├── pages/                # 页面文件
│   ├── home.py           # 首页/仪表盘
│   ├── login.py          # 登录页面
│   ├── page1.py          # 示例页面1（数据管理）
│   ├── page2.py          # 示例页面2
│   └── page3.py          # 示例页面3
├── src/                  # 源代码
│   ├── auth/             # 认证相关
│   │   ├── auth.py       # 认证逻辑
│   │   ├── auth_flow.py  # 认证流程
│   │   └── cookies.py    # Cookie管理
│   ├── config/           # 配置管理
│   │   ├── config.py     # 配置文件
│   │   └── page_config.py # 页面配置
│   ├── navigation/       # 导航配置
│   │   └── navigation.py # 导航管理
│   ├── ui/               # UI组件
│   │   └── sidebar.py    # 侧边栏管理
│   ├── utils/            # 工具函数
│   │   └── logger.py     # 日志工具
│   └── database/         # 数据库相关（新增）
│       ├── models.py     # 数据模型
│       ├── connection.py # 数据库连接
│       └── queries.py    # SQL查询
├── tests/                # 测试文件
│   └── test_utils.py     # 测试工具
├── .env                  # 环境变量
├── .gitignore            # Git忽略文件
├── README.md             # 项目说明
├── requirements.txt      # 依赖管理
└── app.py                # 主入口
```

## 2. 目录结构说明

### 2.1 根目录文件

- **app.py**：应用主入口文件，负责初始化应用、检查登录状态、配置页面和运行主逻辑
- **requirements.txt**：依赖管理文件，记录应用所需的Python包
- **README.md**：项目说明文档，包含项目结构、功能特点、安装指南和开发SOP
- **.gitignore**：Git忽略文件，指定不需要纳入版本控制的文件和目录
- **.env**：环境变量文件，存储敏感配置信息

### 2.2 配置目录

- **.streamlit/**：Streamlit应用的配置目录
  - **config.toml**：Streamlit配置文件，可设置主题、服务器端口等
  - **secrets.toml**：敏感信息存储文件，如数据库连接字符串等

### 2.3 资源目录

- **assets/**：静态资源目录
  - **images/**：图片资源目录，存放应用中使用的图片文件

### 2.4 数据目录

- **data/**：数据文件目录，用于存储Excel、CSV等数据文件
  - 适用于小型应用或原型开发
  - 优点：简单易用，无需额外配置
  - 缺点：性能有限，不适合大规模数据

### 2.5 数据库目录（新增）

- **database/**：数据库文件目录
  - 用于存储SQLite数据库文件
  - 适用于中小型应用
  - 优点：无需额外数据库服务，部署简单
  - 缺点：并发性能有限

### 2.6 日志目录

- **logs/**：日志文件目录，记录应用运行日志
  - **app.log**：应用主日志文件
  - 便于问题排查和系统监控

### 2.7 页面目录

- **pages/**：页面文件目录，每个文件对应一个应用页面
  - **home.py**：首页/仪表盘，显示系统概览和快速入口
  - **login.py**：登录页面，处理用户登录和认证
  - **page1.py**：示例页面1（数据管理），展示数据管理功能
  - **page2.py**：示例页面2
  - **page3.py**：示例页面3
  - 可根据业务需求添加更多页面

### 2.8 源代码目录

- **src/**：源代码目录，包含核心功能模块

  #### 2.8.1 认证模块
  - **auth/**：认证相关模块
    - **auth.py**：认证逻辑，处理用户验证和角色管理
    - **auth_flow.py**：认证流程，处理登录状态初始化和检查
    - **cookies.py**：Cookie管理，处理用户登录状态的持久化

  #### 2.8.2 配置模块
  - **config/**：配置管理模块
    - **config.py**：配置文件，集中管理应用配置
    - **page_config.py**：页面配置，处理页面布局和属性

  #### 2.8.3 导航模块
  - **navigation/**：导航配置模块
    - **navigation.py**：导航管理，定义页面导航结构

  #### 2.8.4 UI模块
  - **ui/**：UI组件模块
    - **sidebar.py**：侧边栏管理，渲染侧边栏内容

  #### 2.8.5 工具模块
  - **utils/**：工具函数模块
    - **logger.py**：日志工具，提供统一的日志记录功能

  #### 2.8.6 数据库模块（新增）
  - **database/**：数据库相关模块
    - **models.py**：数据模型，定义数据库表结构
    - **connection.py**：数据库连接，处理数据库连接和会话管理
    - **queries.py**：SQL查询，封装数据库操作

### 2.9 测试目录

- **tests/**：测试文件目录，存放测试用例
  - **test_utils.py**：测试工具，测试工具函数
  - 可根据需要添加更多测试文件

## 3. 数据库设计考虑

### 3.1 数据库选择

根据应用规模和需求，可以选择以下数据库：

1. **SQLite**：
   - 适合小型应用和原型开发
   - 存储在本地文件中，无需额外服务
   - 推荐存放位置：`database/`目录

2. **PostgreSQL/MySQL**：
   - 适合中大型应用
   - 需要单独的数据库服务
   - 连接信息存储在：`.streamlit/secrets.toml`或`.env`文件

### 3.2 数据库目录结构

#### SQLite数据库：
```
database/
├── app.db              # 主数据库文件
└── migrations/         # 数据库迁移文件
    └── versions/       # 迁移版本
```

#### 关系型数据库（PostgreSQL/MySQL）：
```
database/
├── models/             # 数据模型
├── migrations/         # 数据库迁移
└── seeds/              # 初始数据
```

### 3.3 数据库配置

- **开发环境**：使用SQLite或本地数据库
- **生产环境**：使用远程数据库服务

配置示例（`.env`文件）：
```
# SQLite
DATABASE_URL=sqlite:///database/app.db

# PostgreSQL
# DATABASE_URL=postgresql://username:password@localhost:5432/dbname

# MySQL
# DATABASE_URL=mysql://username:password@localhost:3306/dbname
```

## 4. 模块职责与依赖关系

### 4.1 核心模块职责

| 模块 | 主要职责 | 文件位置 | 依赖模块 |
|------|---------|----------|----------|
| 认证模块 | 用户登录、注销、权限验证 | src/auth/ | config, utils |
| 配置模块 | 集中管理应用配置 | src/config/ | - |
| 导航模块 | 管理页面导航结构 | src/navigation/ | - |
| UI模块 | 渲染侧边栏和UI组件 | src/ui/ | auth |
| 工具模块 | 提供日志等工具函数 | src/utils/ | config |
| 数据库模块 | 数据库连接和操作 | src/database/ | config |

### 4.2 页面与模块关系

| 页面 | 主要功能 | 依赖模块 |
|------|---------|----------|
| 登录页面 | 用户登录 | auth |
| 首页 | 系统概览 | auth, ui |
| 数据管理页面 | 数据CRUD操作 | auth, database |
| 其他页面 | 业务功能 | 根据需求而定 |

## 5. 扩展性考虑

### 5.1 模块扩展

- **新功能模块**：在`src/`目录下创建新的模块目录
- **新页面**：在`pages/`目录下创建新的页面文件，并在`navigation.py`中添加
- **新数据库表**：在`database/models.py`中定义新的数据模型

### 5.2 技术栈扩展

- **前端**：可集成Plotly、Bokeh等数据可视化库
- **后端**：可集成FastAPI、Flask等Web框架
- **数据库**：可从SQLite迁移到PostgreSQL、MySQL等
- **认证**：可集成OAuth、JWT等认证方式

## 6. 最佳实践

1. **模块化设计**：将功能拆分为独立的模块，提高代码可维护性
2. **配置分离**：将配置与代码分离，便于环境切换
3. **异常处理**：对所有可能的异常进行捕获和处理
4. **日志记录**：记录关键操作和错误信息
5. **权限控制**：根据角色控制功能访问
6. **测试覆盖**：为核心功能编写测试用例
7. **代码规范**：遵循Python代码规范（PEP 8）
8. **文档完善**：为模块和函数添加文档字符串

## 7. 总结

本目录结构设计遵循"约定优于配置"的原则，提供了标准化的项目结构和核心功能，便于开发人员快速上手和扩展。通过合理的模块划分和职责分配，确保了代码的可维护性和可扩展性。

同时，考虑了数据库的存放位置和配置方式，支持从简单的SQLite到复杂的关系型数据库的平滑过渡，满足不同规模应用的需求。

这种结构设计不仅适用于Streamlit应用，也可以作为其他Python Web应用的参考模板。
